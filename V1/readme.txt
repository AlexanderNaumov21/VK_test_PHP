Вариант исполнения №1
У нас имеются методы:
1)  POST метод по URL 'http://vktask/api',
 Принимает в body: event_name - имя нашего события, и user_status - статус нашего пользователя либо 0(не авторизированный), либо 1(авторизированный)
Далее заносится всё (+ IP-адрес и дату) в БД.
2) GET метод по URL 'http://vktask/api?counter_type=&counter_value=&sort_tupe=&sort_direction=',
Все атрибуты обязательны для указания.
	counter_type - это выбор критерия вывода информации, доступны 3 значения:
		event - вывод информации по имени события
		ip - вывод информации по ip-пользовотеля
		status - вывод информации по двум группам событий - авторизированные, либо не авторизированные
	counter_value - по какому конкретному значению будет выводится информация: 
		для event - указываем имя события
		для ip - указываем ip-пользователя 
		для status - указываем либо 0(не авторизированный), либо 1(авторизированный)
	sort_tupe - доступны три значения(обязательно нужно указать одно из них):
		event_date - сортировка по дате
		event_name - сортировка по имени события
		none - сортировка не будет произведена
	sort_direction - направление сортировки, доступны три значения:
		ASC - сортирует от низких значений к высоким
		DESC - сортирует от высоких значений к низким
		none - оставляет в дефолтном состоянии
Будет выведен список со всеми данными постов проходящих через выборку.
